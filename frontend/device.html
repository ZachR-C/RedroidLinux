Remote ADB —

Connect the ADB terminal on any computer to this emulator, so tools (Claude Code, Android Studio, scripts) can drive it and test apps.

Step 1 — open an SSH tunnel to the emulator's ADB server keep it running

On the other computer (needs adb + SSH access to the server). Secure — ADB never leaves the server; works on a VPS or through NAT.


      

      
Step 2 — point adb at it & drive the device

      
In another terminal. Any adb tool (Claude Code, Android Studio, scripts) that inherits this env now controls the emulator.

      

      

      
One tunnel exposes every emulator on this server — use the serial above to pick this one. On a VPS, set PUBLIC_HOST/SSH_USER in .env to prefill these.
